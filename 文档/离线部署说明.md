# 离线部署说明

## 适用场景

用于政务、学校、企业内网等无法访问公网的环境。默认离线包包含源码、Docker 编排、部署文档和脚本，不包含第三方大模型、TTS、ASR 的在线服务能力。

## 系统要求

| 项目 | 最低要求 |
|------|----------|
| 操作系统 | Linux（推荐 Ubuntu 22.04 / CentOS 7+）、Windows Server 2019+ |
| CPU | 1 核 |
| 内存 | 2 GB |
| 磁盘 | 1 GB 可用空间 |
| Docker | 24.0+ |
| Docker Compose | v2.20+ |

> 单台 1 核 2G 服务器可稳定承载 100+ 并发会话。

## 打包

在有完整源码的联网机器上执行：

```bash
python scripts/package_release.py
```

产物位于 `release/anti-fraud-v1.0-*.zip`，包含：

```text
├── backend/app/            后端源码
├── backend/requirements.txt
├── backend/Dockerfile
├── frontend/src/           前端源码
├── frontend/public/        静态资源
├── frontend/index.html
├── frontend/package.json
├── frontend/package-lock.json
├── frontend/vite.config.js
├── frontend/Dockerfile
├── frontend/nginx.conf
├── scripts/                部署与运维脚本
├── 文档/                   项目文档
├── docker-compose.yml
└── README.md
```

可通过 `--out` 参数指定输出路径：

```bash
python scripts/package_release.py --out /path/to/output.zip
```

## 部署步骤

### 方式一：目标服务器可访问 Docker Hub

1. 将 zip 拷贝到目标服务器并解压：

```bash
unzip anti-fraud-v1.0-*.zip -d /opt/anti-fraud
cd /opt/anti-fraud
```

2. （可选）配置环境变量：

```bash
cp backend/.env.example backend/.env
# 编辑 backend/.env 按需填入配置
```

3. 启动服务：

```bash
docker compose up --build -d
```

### 方式二：目标服务器完全离线

1. 在联网机器上构建并导出镜像：

```bash
cd /path/to/anti-fraud
docker compose build
docker save anti-fraud-backend:1.0 anti-fraud-frontend:1.0 -o anti-fraud-images.tar
```

2. 将 `anti-fraud-images.tar` 和发布 zip 一起拷贝到目标服务器（U盘、内网传输等）。

3. 在目标服务器导入镜像并启动：

```bash
docker load -i anti-fraud-images.tar
unzip anti-fraud-v1.0-*.zip -d /opt/anti-fraud
cd /opt/anti-fraud
docker compose up -d
```

## 访问地址

| 服务 | 地址 | 说明 |
|------|------|------|
| 前端页面 | `http://<服务器IP>:8080` | 默认端口可通过 `FRONTEND_PORT` 环境变量修改 |
| 后端 API | `http://<服务器IP>:8080/api/` | 通过 nginx 反向代理 |
| API 文档 | `http://<服务器IP>:8000/docs` | 仅后端容器端口直连可访问 |
| 健康检查 | `http://<服务器IP>:8080/api/health` | 返回 `{"status":"ok"}` |

## 环境变量配置

部署时可通过以下方式传入环境变量（按优先级从高到低）：

1. `docker compose` 命令行 `-e` 参数
2. 项目根目录 `.env` 文件
3. `docker-compose.yml` 中的默认值

### 关键配置

| 变量 | 说明 | 离线默认行为 |
|------|------|-------------|
| `DEEPSEEK_API_KEY` | LLM API 密钥 | 空 → 使用内置兜底话术 |
| `TTS_PROVIDER` | 语音合成 | `browser` → 浏览器原生 |
| `ASR_PROVIDER` | 语音识别 | `browser` → 浏览器原生 |
| `JWT_SECRET` | JWT 签名密钥 | 内置默认值（**生产务必修改**） |
| `ADMIN_USERNAME` | 管理员用户名 | `admin` |
| `ADMIN_PASSWORD` | 管理员密码 | `admin123`（**生产务必修改**） |
| `FRONTEND_PORT` | 前端暴露端口 | `8080` |

完整配置项参见 `backend/.env.example`。

## 离线能力边界

| 功能 | 有网络时 | 离线时 |
|------|---------|--------|
| AI 对话 | DeepSeek/通义千问/文心一言 | 内置兜底话术（基于规则引擎） |
| 语音合成 | 讯飞 WebAPI 真人音色 | 浏览器原生 SpeechSynthesis |
| 语音识别 | 讯飞 ASR | 浏览器原生 webkitSpeechRecognition |
| 数据持久化 | SQLite（Docker volume） | SQLite（Docker volume） |
| 管理后台 | 全功能 | 全功能 |
| 场景管理 | 全功能 | 全功能 |

> 离线模式下核心训练流程完整可用，仅 AI 回复质量降级为预设话术。

## 数据持久化

数据通过 Docker named volume 持久化：

| Volume 名称 | 容器路径 | 内容 |
|-------------|---------|------|
| `anti_fraud_data` | `/app/data` | SQLite 数据库文件 |
| `anti_fraud_logs` | `/app/logs` | 应用日志 |

备份数据库：

```bash
docker cp $(docker compose ps -q backend):/app/data/anti_fraud.db ./backup_$(date +%Y%m%d).db
```

恢复数据库：

```bash
docker cp ./backup.db $(docker compose ps -q backend):/app/data/anti_fraud.db
docker compose restart backend
```

```bash
# 查看服务状态
docker compose ps

# 查看日志
docker compose logs -f backend

# 重启服务
docker compose restart

# 停止服务
docker compose down

# 停止并删除数据（慎用）
docker compose down -v
```

## 常见问题

### 前端页面访问白屏

- 确认 `frontend` 容器正常运行：`docker compose ps`
- 确认端口未被占用：`netstat -tlnp | grep 8080`
- 查看 nginx 日志：`docker compose logs frontend`

### 后端接口返回 502

- 确认 `backend` 容器正常运行：`docker compose ps`
- 查看后端日志：`docker compose logs backend`
- 检查健康接口：`curl http://localhost:8080/api/health`

### 语音功能不可用

- 离线环境下语音依赖浏览器原生能力
- 浏览器 ASR 需要 HTTPS 环境（localhost 除外）
- 建议配置 HTTPS 证书或使用 `localhost` 访问
- 如需离线环境下的高质量语音，可考虑：
  - 部署讯飞离线语音 SDK（需额外授权，参见 [讯飞离线包文档](https://www.xfyun.cn/doc/tts/offline_tts/Linux-SDK.html)）
  - 部署开源 TTS 引擎（如 PaddleSpeech、VITS）并将 `TTS_ENDPOINT` 指向本地服务
  - 浏览器 TTS 在 Chrome/Edge 中内置中文语音包，无需联网即可使用

### 数据库损坏

```bash
# 停止服务
docker compose down
# 删除数据 volume
docker volume rm anti_fraud_data
# 重新启动（自动初始化新数据库）
docker compose up -d
```

## 升级

1. 备份当前数据库。
2. 用新版本 zip 替换源码目录。
3. 重新构建并启动：

```bash
docker compose down
docker compose up --build -d
```

数据库结构会在启动时自动迁移（`init_db()` 负责增量建表）。
