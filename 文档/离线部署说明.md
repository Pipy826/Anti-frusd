# 离线部署说明

## 适用场景

用于政务、学校、企业内网等无法访问公网的环境。默认离线包包含源码、Docker 编排、部署文档和脚本，不包含第三方大模型、TTS、ASR 的在线服务能力。

## 打包

在有完整源码的机器上执行：

```bash
python scripts/package_release.py
```

产物位于 `release/anti-fraud-v1.0-*.zip`。

## 部署

1. 将 zip 拷贝到目标服务器并解压。
2. 如目标服务器可访问 Docker Hub，执行：

```bash
docker compose up --build -d
```

3. 如目标服务器完全离线，应提前在联网机器构建并导出镜像：

```bash
docker compose build
docker save anti-fraud-backend anti-fraud-frontend -o anti-fraud-images.tar
```

在离线服务器导入：

```bash
docker load -i anti-fraud-images.tar
docker compose up -d
```

## 离线能力边界

- 无外网时，大模型、第三方 TTS、第三方 ASR 会自动降级。
- 未配置 `DEEPSEEK_API_KEY` 时，后端使用内置兜底话术。
- 未配置 `TTS_ENDPOINT` / `ASR_ENDPOINT` 时，前端使用浏览器 TTS / ASR。
- SQLite 数据和日志通过 Docker volume 保存。
