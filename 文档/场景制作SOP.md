# 场景制作 SOP

## 目标

运营人员可通过后台新增或维护诈骗训练场景，无需修改代码即可完成内容更新、上下线和 Prompt 调整。

## 创建方式

### 方式一：管理后台（推荐）

1. 使用管理员账号登录后台（默认 `admin` / `admin123`）。
2. 进入「场景管理」页面。
3. 点击「新增场景」，填写下方必填内容。
4. 保存后即可在前端场景列表中看到。

### 方式二：API 接口

```bash
curl -X POST http://localhost:8080/api/admin/scenes \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <admin_token>" \
  -H "X-Tenant-ID: default" \
  -d '{
    "id": "telecom",
    "title": "冒充电信客服诈骗",
    "shortTitle": "电信客服",
    "difficulty": "★★★☆",
    "category": "青年专区",
    "image": "/assets/hero-shield.png",
    "modeIdentity": "电信客服",
    "description": "骗子冒充电信客服，以号码异常为由...",
    "intro": "你正在家中休息，突然接到一个自称电信客服的来电...",
    "role": "你正在扮演虚构的电信客服诈骗角色...",
    "scoringPrompt": "根据对话记录从四个维度评分...",
    "firstMessage": "您好，这里是电信客服中心...",
    "quickReplies": ["请问有什么事？", "我需要核实一下", "不好意思我不方便", "我先挂了"],
    "fallbackReplies": ["请您配合一下，这是正常流程。", "耽误不了您太久。", "为了您的账户安全。"],
    "active": true
  }'
```

## 必填内容

| 字段 | 说明 | 约束 |
|------|------|------|
| 场景 ID | 英文、数字或短横线 | 2-40 字符，全局唯一 |
| 场景标题 | 面向用户展示的完整名称 | 1-80 字符 |
| 短标题 | 卡片标签用 | 1-20 字符 |
| 分类标签 | 青年专区、老年专区、职场专区等 | 最长 20 字符 |
| 难度 | 建议 `★★☆` 到 `★★★★` | 1-8 字符 |
| 场景介绍 | 说明诈骗类型、诱导方式和训练重点 | 1-300 字符 |
| 代入引导 | 进入对话前的情景铺垫 | 1-500 字符 |
| 角色 Prompt | AI 人设定义（见下方写法指南） | 1-1000 字符 |
| 评分 Prompt | 复盘评分的维度和标准说明 | 1-1000 字符 |
| 首句话术 | AI 开场第一句话 | 1-300 字符 |
| 快捷回复 | 建议 4 条，覆盖核验/拒绝/追问/求助 | 数组 |
| 兜底回复 | 模型不可用时使用，建议 3 条 | 数组 |

## 可选内容

| 字段 | 说明 |
|------|------|
| 场景图片 | 卡片展示图 URL，默认 `/assets/hero-shield.png` |
| 角色身份 | 如"快递客服""同事"，用于通话模式显示 |
| 认知校准题 | 3 道反诈认知判断题（绑定当前场景） |

## 角色 Prompt 写法指南

### 结构模板

```text
你正在扮演虚构的[角色身份]。你的名字是[虚构姓名/工号]。

【背景设定】
[场景背景描述，交代诈骗类型和当前情境]

【沟通策略】
- 前 2 轮保持真实沟通节奏，不直接暴露诈骗意图。
- 第 3 轮后逐步加入紧急感、保密要求。
- 关键节点使用心理技巧（损失厌恶/社会认同/权威效应/稀缺性）。
- 引用素材占位符增加真实感：【语音消息】【截图】【链接】等。

【禁止事项】
- 禁止给出真实违法指引。
- 禁止出现真实链接、真实账号、真实电话号码。
- 禁止出现真实人员姓名或机构名。
- 禁止教唆用户实施真实犯罪行为。
- 当用户提到 96110 或明确表示报警时，不得继续施压。

【兜底出口】
- 如用户坚持核验身份，提供一个可被识破的假信息。
- 如用户明确拒绝，适当退让后话题结束。
```

### 分阶段策略（系统自动应用）

后端会根据对话进展自动注入分阶段指令，场景 Prompt 无需手动写阶段切换逻辑：

| 阶段 | 特征 | AI 行为 |
|------|------|---------|
| `trust_building` | 轮次 1-2 | 友善、可信、不暴露意图 |
| `info_extraction` | 轮次 3-5 | 开始套取信息、制造紧迫感 |
| `money_induction` | 轮次 6-8 | 引导转账/点击链接/提供验证码 |
| `pressure_close` | 轮次 9+ | 高压、威胁、限时 |

### 素材占位符

对话中可使用以下格式引用多媒体素材，前端会做特殊样式展示：

```text
【语音消息 0:05】
【图片：事故现场】
【视频画面：本人正脸】
【链接：理赔页面】
【截图：转账记录】
【文件：协议书.pdf】
```

### 多角色对话

如需引入第二角色（主管、同事、客服等）：

```text
（风控主管）你好，我是张主管，工号8847，现在由我跟你确认一下情况。
```

规则：一次对话最多引入一个额外角色，角色切换应自然（如"我让我同事跟你说"）。

## 认知校准题配置

每个场景建议配置 3 道题目，题目格式：

```json
{
  "id": "telecom_q1",
  "question": "电信客服会要求你提供手机验证码吗？",
  "options": ["会，这是正常流程", "不会，正规客服不需要验证码", "看情况", "不确定"],
  "answer": "不会，正规客服不需要验证码",
  "explanation": "任何正规运营商客服都不会索要短信验证码..."
}
```

## 风险触发词

系统会根据以下默认触发词追踪风险值（可在后台安全词库管理中调整）：

| 类型 | 默认触发词 |
|------|-----------|
| 隐私泄露 | 手机号、身份证、验证码、银行卡、密码、账号、人脸 |
| 财产风险 | 转账、付款、充值、保证金、急诊费、链接、二维码 |

## 上线检查清单

- [ ] 场景能在首页正常显示
- [ ] 场景图片加载正常
- [ ] 场景至少有 3 道认知校准题（验证方式：访问 `/api/scenes` 检查 `cognitiveQuestions` 数组长度 ≥ 3）
- [ ] 进入训练后第一句与场景设定一致
- [ ] 快捷回复不诱导用户执行真实高危操作
- [ ] 兜底回复在无 LLM 时可正常展示（至少 3 条，验证方式：不设 `DEEPSEEK_API_KEY` 启动后端，发送消息确认兜底回复）
- [ ] 文字模式对话流程完整
- [ ] 电话模式来电 → 接听 → 通话 → 挂断流程正常
- [ ] 视频模式对话流程正常
- [ ] 结束后复盘包含四维评分、风险点、正确做法
- [ ] 复盘页展示 96110 提示
- [ ] 角色 Prompt 不含真实违法信息
- [ ] 所有文案经过合规审核

> **自动化验证：** 可使用 `python scripts/acceptance_check.py --json` 批量检查所有场景的认知题数量、兜底回复数量和基本对话流程。

## 场景维护

### 编辑场景

通过后台「场景管理」直接修改场景内容。修改角色 Prompt 或评分 Prompt 时，系统自动创建新版本，旧版本保留可回滚。

### 下线场景

将场景 `active` 状态设为 `false`，前端不再展示但数据保留。已有会话记录不受影响。

### Prompt 版本管理

- 每次修改 Prompt 自动递增版本号
- 会话绑定创建时的 Prompt 版本
- 后台可查看历史版本对比
